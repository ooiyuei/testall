## Executive Summary

**Testall**は、模試・校内テスト結果を起点に、**受験本番から「次の25分」までを6つの時間軸で逆算する**受験戦略OS。

世の中の学習サービスが「今やる勉強」だけを最適化する中、Testallは **本番 → 1年 → 3ヶ月後の模試 → 今週 → 今日 → 次の25分** を一気通貫で繋ぐ唯一のプロダクト。

**ターゲット**: 偏差値55-60の「続かない楽観的な高2」 — 推定58万人、TAM 209億円（中学・浪人・社会人含めると2,100億円）。

**ビジネスモデル**: 月額1,500円のサブスク。LTV 21,000円 / CAC 2,000円で **LTV/CAC = 10.5倍**。3年目に5万人で年商15億円規模、シリーズAバリュエーション50-80億円を狙う。

**最終ミッション**: **親の年収で、受験を諦めさせない世界をつくる。**

## I. ミッション・ビジョン・哲学

### ミッション

**親の年収で、受験を諦めさせない。**

塾代年100万円を払える家庭と払えない家庭で、東大合格率は約4倍違う。この構造的格差を、AIによる個別最適教育の低価格化で構造的に解消する。

### ビジョン（2035年）

**日本中の受験生が、AIマネージャーと共に「次の25分」を生きている。**

塾の有無、地域、家庭の経済状況に依らず、すべての受験生が最高の伴走者を持つ。日本の受験は「家庭環境」ではなく「努力の質」で決まる社会にする。

### プロダクトの3つの哲学

1. **気合いではなく、仕組みで続く** — モチベーション依存をやめる
2. **時間ではなく、行動の質を測る** — 勉強時間マウントを終わらせる
3. **答えではなく、自力で解ける状態を作る** — 分かった気を壊す

## II. 解くべき本質課題

### 課題1: 高校生は、自分をマネジメントできない

- スマホという最強の誘惑が常に手元にある
- モチベーションは波打ち、安定しない
- 気合いで乗り切ろうとして、テスト前に徹夜して燃え尽きる
- 「焦り → 徹夜 → 放置 → また焦り」のサイクル

### 課題2: 努力が、成果に変換されていない

- 高校生の平均学習時間：1日2.6時間（ベネッセ2023）
- 模試後3日以内に振り返る生徒：推定2割以下
- 「何をやればいいかわからない」と答える受験生：6〜7割
- 学習アプリ4週間継続率：推定10%以下（スタプラ・デュオリンゴ等）

### 課題3: 受験戦略を立てる伴走者がいない

- 集団塾は高い（年100万円）、行ける家庭が限られる
- 親も学校の先生も「具体的に何をやれ」までは指示できない
- ChatGPTは聞かれたことしか答えない、計画は立てない
- 結果、塾代を払えない家庭と払える家庭で東大合格率4倍差

### 既存解の限界

| サービス | 何を解いているか | 限界 |
|----------|-----------------|------|
| 集団塾 | 場所・授業・ペース | 個別最適なし、高額 |
| 個別指導塾 | 個別の質問対応 | 高額、講師の質バラつき |
| スタディサプリ | 授業動画 | 「何をやるか」は決めない |
| Studyplus | 学習記録 | 記録だけ、行動は変わらない |
| 武田塾型 | 参考書ベース計画 | 高額、属人的 |
| Duolingo | ゲーム化継続 | 受験には特化していない |
| ChatGPT | 質問対応 | 戦略・計画は作れない |
| atama plus | AI個別問題演習 | 塾経由のみ、戦略レイヤーなし |

**誰も「テスト結果 → 今日の25分」までを繋いでいない。**

## III. インサイト：誰も解いていない問題

### 核心インサイト

**世の中の学習サービスは「今やる勉強」だけを最適化している。**

- タイマーアプリ → 25分という実行単位を最適化
- Studyplus → 記録を最適化
- 塾 → 今日の授業を最適化
- 参考書 → 1冊の中身を最適化
- atama plus → 1問単位の最適化

しかし受験生に必要なのは、**「本番という遠いゴール」から「次の25分」までを繋ぐ逆算の設計**。

### なぜ既存プレイヤーは解けないか

1. **塾**: 1人の講師が60人の全教科・全テストを毎日逆算するのは物理的に不可能
2. **タイマーアプリ**: そもそも戦略を持たない、実行だけ
3. **AIチャット**: 受験生が「自分の状況」を毎回入力するコストが高すぎる
4. **学習記録アプリ**: 記録 → 分析 → 次の行動のループを閉じていない
5. **atama plus等**: 個別問題演習はやるが、本番までの逆算戦略は塾講師頼み

**AIが、テスト結果という最強の入力データから、本番までの全時間軸を自動で繋ぎ続ける** — これが2024年以降にようやく可能になった、誰も解いていない問題。

## IV. ソリューション：逆算設計OS

### 一行定義

**Testallは、本番から、次の25分までを逆算する受験戦略OS。**

### 6つの時間軸を繋ぐ

```
   受験本番（合格）
     ↓ 残り○ヶ月
   1年後の自分（合格ラインへの差分）
     ↓ 残り○模試
   3ヶ月後の模試（1Qマイルストーン）
     ↓ 残り○週
   今週やること（5教科の配分）
     ↓ 残り○ブロック
   今日やること（25分 × N個）
     ↓
   次の25分でやる1問
```

模試を撮るだけで、この6階層が **毎朝自動で再計算** される。

### 2つの実行単位

**戦略単位 = 逆算設計**
- 本番からの残期間、苦手単元、所有参考書の進捗から最適な学習配分を毎日再計算
- 計画が崩れた日も翌朝AIが再設計するので、燃え尽きない

**実行単位 = 25分ブロック**
- ポモドーロ × 受験計画。「人間が実行できる最小単位」
- 完了条件「○問を答え見ずに解ける」で「分かった気」を壊す
- 45分ではなく25分にした理由：続かない高校生にとって25分が「適度」の閾値

戦略と実行の両方が揃って初めてOS。タイマーアプリは実行のみ、塾は戦略のみ。**Testallは両方。**

## V. プロダクト

### コア体験ループ

```
   テストやる
       ↓
   どこが苦手か わかる
       ↓
   何からやればいいか わかる
       ↓
   適度に頑張って 続けられる
       ↓
   成績が 伸びる
       ↓
   勉強が 楽しくなる
       ↓
   （次のテストへ：ループ最初に戻る）
```

このループが、ターゲットの「続かない高校生」を「続く高校生」に変える。

### ユーザージャーニー

**Day 0: オンボーディング（5分）**
- 志望校（未決定でもOK）・現在の偏差値・所有参考書・使える時間を登録
- 本番までの残期間と必要伸び幅をAIが算出

**Day 1: 初回テスト登録（3秒）**
- 模試 or 校内テストを撮影
- AIが30秒で苦手単元・ミス原因を構造化

**Day 1: 逆算計画生成（自動）**
- 3ヶ月計画 → 1ヶ月計画 → 1週間計画 → 今日のタスク → 次の25分

**Day 1〜: 25分ブロック実行**
- 例: 17:00〜17:25「数学：二次関数 例題12〜15」
- 集中モード起動、スマホ封印
- 完了条件：4問中3問を答え見ずに解ける
- 終わったら自己評価1タップ

**Day 2: 自動再設計（毎朝6時）**
- 昨日の進捗から計画を毎朝再計算
- 「できなかった」も「できた」も自動反映

**Week 4: 進捗フィードバック**
- 「先週より英文法の正答率が15%上がった」
- 成績が伸びる体験 → 勉強が楽しくなる

### プロダクトの5つの原則

1. **入力は3秒以下** — 続かない人を続けさせる絶対条件
2. **指示は具体的に1つ** — 「何やればいい？」を残さない
3. **崩れた前提で設計** — できなかった日が標準
4. **完了条件は自力再現** — 「分かった気」を許さない
5. **他人と比較しない** — 昨日の自分とだけ比べる

### 技術スタック

| レイヤ | 技術 | 役割 |
|--------|------|------|
| フロント | Next.js / React | スマホファーストUI |
| バックエンド | Supabase | DB・認証・ストレージ |
| AI: OCR | Claude Vision | テスト画像から問題抽出 |
| AI: 単元分類 | Claude Sonnet | 問題→単元・難易度マッピング |

| 課金 | Stripe | サブスク管理 |

3. **塾を選ばない層** — 我々の最大の競合（塾）を最初から避けている
4. **アプリリテラシーが高い** — スマホファーストUIに馴染む

### 拡張ターゲット（段階展開）

- **Phase 2 (Y2-)**: 高1・浪人生・難関校志望の偏差値60-70層
- **Phase 3 (Y3-)**: 中学生（中高一貫・高校受験）

### 顧客の動機（Jobs to be Done）

2. **感情ジョブ**: 「続かない自分」から脱したい、自己効力感が欲しい
3. **社会ジョブ**: 親や友達に「ちゃんとやってる」と認められたい

## VII. 市場機会

### TAM / SAM / SOM（ボトムアップ）

```
TAM (Total Addressable Market)
   高校生 290万人 × 独学志向 20% = 58万人
   × 月1,500円 × 24ヶ月 = 209億円

   + 中学・浪人・社会人学習 = 2,100億円
```

```
SAM (Serviceable Available Market)
   都市部高校生（人口集中地区） 200万人
   × 独学志向 20% = 40万人
   × ARPU 30,000円（生涯） = 120億円
```

```
SOM (Serviceable Obtainable Market) - 3年目
   有料5万人 × 30,000円 = 15億円/年
```

### 市場成長ドライバー

- **個別最適教育の制度化** — 学習指導要領改訂で「個別最適な学び」が国策に
- **塾離れ・自学志向** — コロナ後Studyplusユーザー400万人、独学派増加
- **AI技術の現実化** — Claude/GPTで個別計画生成が実用段階
- **教育格差の社会問題化** — 親ガチャ・親の年収議論が政策レベルに
- **少子化下の単価上昇余地** — 1人あたり投資額は増加傾向

## XII. 防御戦略（Moat）

### 三層の濠

**1. データ濠 — 逆算アルゴリズムの精度**

蓄積するデータ：
- 模試 × 単元 × ミス原因の構造化
- 残期間 × 目標偏差値 × 進捗の組み合わせ別の最適計画
- 計画の崩れパターンと最適なリカバリー手段

これは時間でしか作れない。1年早く始めれば、競合は永遠に追いつけない。

**2. 教材マッピング濠**

- 主要参考書500冊 × 単元 × 難易度 × ページ番号のマッピング
- 「この苦手にはこの参考書のこのページ」を提案できる唯一のサービス
- 人力構築コストが新規参入の壁

**3. ミッション濠**

- 「親の年収で受験を諦めさせない」というメッセージ
- 自治体・公教育の連携入口になる
- 営利オンリーの競合は入れない領域

### ネットワーク効果

- ユーザーが増える → データが増える → 計画精度が上がる → ユーザー満足度上がる → 口コミ拡散 → ユーザーが増える
- 各都道府県・学校でローカルデータが蓄積（「○○高校の生徒は△△で詰まる」）
- 学校単位での口コミ伝播（同じ学校で2人使い始めると指数関数的に広がる）

### 著作権・規制への対応

- 問題文の中身は保存しない、構造化メタデータ（単元・難易度・正誤）のみ蓄積
- 参考書はページ番号と問題番号のみ管理、本文は参照しない
- 学生所有教材のみ計画対象（「持っている」と申告した教材のみ）
- 出版社提携を段階的に展開（汎用ペース提案 → 後に公式連携で精度向上）

## XVII. 社会的インパクト

### 解決する社会課題

**親の年収100万円差で、東大合格率は約4倍違う。**

- 塾代年100万円を払える家庭 ≒ 約25%
- 払えない家庭 ≒ 約75%
- この格差は受験結果に直結し、人生機会の格差に連鎖する

### Testallの構造的解決

1. **価格破壊**: 塾の1/20でAI個別最適教育を提供
2. **場所制約解消**: 地方在住者も都市部と同じ伴走を受けられる
3. **自治体連携**: 公立高校への無償提供で経済的弱者にも届ける
4. **データ駆動**: 教育格差の構造を匿名データで可視化し、政策提言に活用

### 長期ビジョン（2035年）

**日本の受験は「家庭環境」ではなく「努力の質」で決まる社会へ。**

- 全高校生の30%がTestallユーザー
- 公教育との連携で経済的弱者にも標準サービスとして提供
- 教育格差の指標（家庭年収別合格率の差）が縮小

### 真のKPI

売上規模ではなく、**何人の高校生に最高の伴走者を渡せたか**で測る。
「親の年収で諦めなかった子」の数が真のKPI。
