---

## v2.0 → v2.1 で何が変わったか

v2.0 は構造（問題フェーズの長さと転換点の位置）を直したが、聴衆ヒアリングで6つの問題が浮上：

| # | 問題 | v2.1 での対応 |
|---|------|--------------|
| ① | 問題が抽象的（「何をやればいいか」では何のことか不明） | 「差分・逆算・落とし込み」の3層に分解して3スライド化 |
| ② | Slide 07「本人のせいじゃない」と Slide 08「翌日に見直したのは5人」が矛盾 | データを「行動の証拠」から「構造の証拠」に書き換え |
| ③ | 転換点「答えがある」が弱い | 「テストは答え合わせじゃない。地図だ」に格上げ |
| ④ | 6つの時間軸がフィーチャー説明 | 3つの痛みに1対1で答える形式に再構成 |
| ⑤ | ビジネスフェーズが6スライドで詰まりすぎ | 4スライドに統合（LTV+WhyNow / Moat+Traction） |
| ⑥ | クロージングが冒頭タグラインと呼応していない | 「家庭教師が、手のひらの中にある」を結末で回収 |

| 「6つの時間軸って、計画アプリと何が違う？」 | 計画アプリは「ユーザーが入力した計画」を管理するだけ。Testallは「ユーザーの実力データから逆算して計画自体を生成」する。入力ゼロが本質的な差。 |
| 「テストは地図、って具体的に何が見える？」 | 例えば「現代文の二項対立問題で3問落とした」場合、ゴール側（共通テスト第1問）との距離が出て、参考書のどの章を何時間やればいいかが今日のタスクになる。 |
| 「差分・逆算・落とし込みは塾も同じでは？」 | 塾の差分は週1〜月1の頻度。Testallはテスト1枚撮るたびに更新される。頻度と継続性が桁違い。 |
| 「写真からどこまで読める？」 | 答案の○×・記述部分・点数を読み、設問IDと照合して単元×ミスタイプに分類する。手書きでも実用域。 |
