# Testall ピッチ v3.0（5分 / スタートアップ大人基準）

**最終更新**: 2026-05-26
**前提**: docs/BUSINESS_PLAN_v3.0.md と整合
**想定**: 5分ピッチ・大人スタートアップ基準（学生ビジコン以上、ICCカタパルト/B Dash Camp相当）

---

## 設計思想

3つの参照源を統合：

1. **Gymee（過去ピッチ）** — 質問ナラティブ / 大きな日本語タイポ / 暗背景の感情ピーク / 棒グラフ価格破壊 / 同心円TAM / 創業者ストーリーは最後
2. **世界・国内成功ピッチ** — Airbnb（Problem-Solution段階分け）、ログラス（700分練習、1スライド1ポイント）、Sequoia（10要素）、SmartHR宮田（Problemに時間を割く）
3. **Testall v3.0事業案** — 本番→25分の逆算設計OS / 続かない高2マス層 / 気合いより仕組み / コアループ

### デザイン原則（守るべき7つ）

1. **1スライド = 1メッセージ**（30字以内、フォント72pt+）
2. **質問で巻き込み、断定で殴る**（Gymee式ナラティブ）
3. **暗背景スライドで感情ピーク**（「もったいない」「次の25分だけ勝て」）
4. **巨大な数字で記憶残し**（2.6h, 4倍, 58万人, 5.4万円）
5. **明色背景基調**（#F5F5F0 cream or #E8EDF3）、強調時のみ黒背景
6. **創業者ストーリーは最後**（写真込み・原体験）
7. **1分300-350字、5分=1,500字目標、最低50回練習**

# デリバリー設計

## 声と話速

- **5分=1,460字**を意識。原稿を声に出して**ストップウォッチ計測**
- **第一声を大きく**。最後尾の聴衆に届かせる
- **重要文の前後で0.5〜1秒の「間」**。沈黙は武器
- 「次の25分だけ勝て」「親の年収で諦めさせない」は**ゆっくり、強く**

## 目線（Zの法則）

- 後方左 → 前方右 → 前方左 → 後方右 で会場を見渡す
- 各スライドで1人ずつ審査員と目を合わせ、1秒キープ
- **暗背景スライドではスクリーンを見ない**（観客のみ）

## 身体・ジェスチャー

- 動きは往復1回程度。歩き回りNG
- 手は胸より上で、大きくゆっくり
- 核となる単語で身体を使う：
  - 「6つの時間軸」→ 指で6つを示す
  - 「3層の濠」→ 手で3層を示す
  - 「変換効率」→ 両手で変換のジェスチャー
  - 「親の年収で諦めさせない」→ 聴衆を見渡しながら

## 練習計画
